I am a 2nd year software engineering student who is passionate to learn more about mobile app development! In this subject, I developed an app called Grocerlyst which helps users manage their shopping items, track dynamic weekly budget limits in real-time, scan barcodes, and sync data seamlessly to cloud platforms.

SDG 12: Responsible Consumption & Production

SDG Statement: Many people struggle to track their grocery spending in real-time, leading to overspending and food waste. Grocerlyst provides a simple tool to log items and monitor a weekly budget to encourage more mindful shopping habits.

Lab Exercises (1 - 5)

Lab 1: UI Implementation

Cloning the layout static architecture of a grocery budget application using fundamental UI Composables.

Reflection: I learned how to use Box layouts as root scopes to cleanly layer background visual components. Nesting organized Columns and Rows properly taught me the strictness of Jetpack Compose scope alignment trees.

Lab 2: User Interaction & State

Implementing runtime form captures, active dynamic input elements, and handling input soft keyboard focus trees.

Reflection: This exercise detailed state variables initialized via remember and mutableStateOf patterns. I successfully engineered dynamic validation prompts and leveraged focus settings gracefully.

Lab 3: Material Design Theme

Generating a responsive application theme using Material 3 color palettes and applying fluid transitions.

Reflection: I mastered programmatic Material Theme color mapping algorithms across light and dark system backgrounds to improve visual hierarchy.

Lab 4: Multi-Screen Navigation

Structuring a decoupled application flow via standard Navigation Controllers and persistent view state architectures.

Reflection: Transitioning local component states into centralized ViewModel classes proved how isolated data layers effectively survive device orientation changes.

Lab 5: Local Database Persistence

Implementing permanent local cache layers using asynchronous Room Data Access Object operations.

Reflection: This lab focused on offline-first storage mechanics using explicit Room entities and repository interfaces to securely handle local data tables.

Overall Learning Journey Reflection

My journey from building a simple static design layout clone in Lab 1 to deploying an asynchronous, multi-layered cloud-synced system in Project 2 has completely transformed my understanding of modern mobile system design. Overcoming early layout syntax challenges built the foundational debugging skills needed to tackle complex, stateful asynchronous workflows later in the course.


Integrating hardware camera sensors alongside network call handlers like Retrofit and continuous Firestore cloud database snapshots demonstrated the true power of reactive programming. This entire educational milestone has equipped me with the skills to confidently build production-ready mobile apps that combine local device utilities with cloud data infrastructures to effectively address pressing global needs like SDG 12.
